Доп. методы отключения не нужных приложений

Название приложения: PatchWall
Статус: можно отключать, но по этой инструкции
Имя пакета: com.miui.tvhome.atv
Расположение исходного APK: /system/priv-app/TvHome/
Отключение PatchWall.
В настройках устройства - выбрать PatchWall- в графе "область обслуживания" выставить регион "другой источник".
После чего приставка перезагрузится.
Затем, настройки -- приложения -- и отключить PW
После таких манипуляций на кнопку mi автоматом подвязывается live tv, т.е эфирные каналы.
Так же можно отключить уведомления и запретить все доступы. Отключить лаунчер по умолчанию.
У меня тв 32" . Думаю с другими tv, будет аналогично.
Описание: Лаунчер от Xiaomi с рекомендациями. На мой взгляд сильно перегруженный всякими рекомендациями, в связи с чем вызывает торможения.

Название приложения: TvServices
Статус: можно отключать, если очень хочется
Имя пакета: com.mediatek.tv.service или com.mediatek.tv.agent
Расположение исходного APK: /system/priv-app/TvServices/
Описание: Отвечает за самодиагностику ТВ, которая автоматически запускается в 5:00 утра, пробуждая телевизор из режима сна. Также отвечает за таймер отключения в настройках ТВ и многое другое. Подробнее о этом приложении писал ЗДЕСЬ.
Недостатки, с которыми столкнулись на практике, после отключения приложения:
    действия по отключению приложения TvServices (com.mediatek.tv.service) ломает таймер сна из настроек телевизора. Какое бы время отключения я не выбирал, телевизор не засыпает после выбранной настройки. Но есть какое-никакое обходное решение: работает таймер из настроек Заставка. Там можно выбрать в качестве заставки Сон и задать временной интервал (придется сделать несколько лишних нажатий, чтобы добраться до этих настроек).
    если используете сторонний лаунчер (например FLauncher), то в нем могут перестать грузиться иконки приложений. В стандартном лаунчере Android TV Home такой проблемы нет.



Перекомпиляция для ускорения запуска и работы ТВ
После подключения к телевизору, вводите эти команды:

adb shell pm compile -a -f --check-prof false -m everything
adb shell pm compile -a -f --check-prof false --compile-layouts
adb shell pm bg-dexopt-job

В результате мы получаем быструю загрузку, мгновенный отклик, стабильную работу приложений.
Вернуть назад настойки вы можете введя эти команды:

adb shell pm compile -a -f -r install
adb shell pm bg-dexopt-job

Отменить изменения для конкретного приложения
adb shell pm compile -f -r install -имя пакета, например com.teamsmart.videomanager.tv-

Для того чтобы включить эту функцию надо ввести три команды, выделенные красным цветом:
adb shell settings put global cached_apps_freezer enabled
Команда имеет три значения: enabled|disabled|default, по умочанию у нас default-отключена
adb shell device_config put activity_manager_native_boot use_freezer true
Чтобы смягчить включение функции дадим ей немного времени
adb shell device_config put activity_manager_native_boot freeze_debounce_timeout 1000
adb reboot
